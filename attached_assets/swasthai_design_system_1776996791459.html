Why Forest Green? Indian healthcare is personal. The green evokes trust, life, Ayurveda tradition, and calm — not cold clinical white. The warm cream background feels like paper, familiar to a country that grew up with paper records. Amber signals attention without panic. Red is reserved only for truly critical values — if everything is red, nothing is.

Trust, UX & Design Principles
What makes a healthcare app feel trustworthy? Every pixel choice communicates safety or danger. Here's the philosophy.
🔒
Explicit Privacy, Everywhere
Every screen where patient data appears shows a lock icon and "Your data stays private." Health data trust is built through constant visible reassurance — not just a one-time terms page.
🎯
Plain Language First
"HbA1c worsened" not "glycated hemoglobin A1c longitudinal deterioration detected." Patients are not doctors. Every value, every alert, every action must be expressed the way a caring friend would say it.
⚠️
Calibrated Urgency
3-tier alert system. Red = critical action NOW (5% of alerts). Amber = attention needed soon (25%). Green = stable (70%). If everything screams, nothing registers. Most people need calm reassurance, not panic.
👆
Mobile-First, Touch-First
All tap targets minimum 44px. Thumb-reachable navigation bar. Large readable fonts (minimum 12px for data, 14px for body). Works on a 4-year-old Android phone on 3G. India is mobile.
Design Principles for Indian Healthcare Context
Multilingual by default. UI strings in Hindi + English mixed. Lab names stay in English (HbA1c is HbA1c in any language). Instructions and alerts in Hindi/regional language. Never ask user to "switch language."
Low-literacy compatible. Color + icon + number + text = 4-layer redundancy for every critical value. A patient who can't read well can still understand "red = problem, green = okay" from the traffic-light system.
Doctor-respectful language. SwasthAI never says "you have condition X" or "you should do Y." Always: "Your doctor may want to discuss..." or "Ask your doctor about..." This is a memory aid, not a diagnostic tool.
Family-aware architecture. The product assumes shared decision-making. Multiple family members see the same health workspace. Caregiver and patient are equally important personas.
Forgiveness-first interactions. Every upload can be edited. Every extracted value can be corrected. Every alert can be dismissed with a reason. The AI is a starting point, not a verdict.
Progressive disclosure. Dashboard shows 4 numbers. Timeline shows details. Report shows full extraction. Never overwhelm — patient first sees only what matters most.
Offline resilience. Core dashboard loads from cached data on poor connectivity. Upload queues locally and syncs when connection returns. India's rural connectivity cannot be assumed.

If that sentence is true, everything else follows. If it's false, no amount of beautiful design saves the product.
