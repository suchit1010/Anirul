Patient Portal

The primary product. Mobile-first, works on 3G, Hindi+English UI. Feels like a trusted family health companion, not a medical system.
1. Onboarding Flow
1
Sign up with phone (OTP)

Profile setup

3
Optional: Link ABHA ID

4
First upload prompt
"Upload any old report to get started" — guided upload wizard with camera shortcut for phone users.

WhatsApp bot connect

Doctor Portal
Doctor Portal — Full Feature Spec
Separate login. Clinic-branded. Shows only what patients consent to share. The single goal: save 5–8 minutes per consultation by giving doctors context before the patient sits down.
Key insight: Doctors don't want another app. They want a 1-page brief before the patient walks in. Everything in this portal is designed around that single need.
1. Doctor Onboarding
1
Sign up with medical registration number
MCI/NMC number verification. Clinic name, specialty, address. Phone OTP for identity.
2
Clinic workspace setup
Clinic name, logo, working hours, appointment duration (15/20/30 min). Multiple doctors can join one clinic.
3
Patient invite link
Doctor gets a unique link to share with patients: "swasthai.in/join/dr-mehta" — patient clicks → links their account to this doctor → consents to share records.
2. Today's Patient Queue
Appointment list
Chronological today's appointments. Patient name, age, condition tags, time slot, brief generation status.
Brief ready indicator
"Brief ready" green badge when AI pre-consult brief has been generated for that patient. Tap to open.
Patient urgency flags
Red dot if patient has a critical alert (e.g., HbA1c >9 flagged since last visit). Doctor sees this before entering the room.
Quick patient search
Search by name, phone, ABHA ID. Works across all patients who have shared records with this doctor.
app/doctor/queue/page.tsxcomponents/doctor/patient-queue.tsx
3. Pre-Consult AI Brief (Core Feature)
1-page summary
Auto-generated by Claude API 15 minutes before appointment. Reads from patient's consented timeline. Shows: patient line, urgent flags, recent labs, medications, trends, suggested discussion points.
Specialty-aware
Endocrinologist brief focuses on glycemic data. Cardiologist gets BP trend + ECG notes. GP gets broad summary. Inferred from doctor's specialty at signup.
Confidence markers
Each extracted value shows confidence level. Low-confidence values marked "⚠ verify against original." Doctor always has final say.
Shareable link
Brief accessible via one-time URL (24h expiry). Doctor can open on phone, tablet, or desktop. No SwasthAI login required for brief-only access.
Offline PDF
Download 1-page PDF brief for offline consultation. Clinic letterhead option for premium plans.
4. During Consultation
Voice-to-text notes
Doctor speaks consultation notes. Claude transcribes + structures into: diagnosis update, new medications, follow-up instructions. Auto-syncs to patient timeline with doctor's tag.
Patient timeline (read-only)
Full patient health timeline visible during consultation. Doctor can reference any past report, medication, or value.
Add prescription
Type new prescription → auto-checks interactions with patient's existing medications → adds to patient timeline → patient gets WhatsApp copy.
Order lab tests
Select tests from list → auto-checks if same test done recently → creates care task on patient's app → WhatsApp reminder when due.
5. Follow-up Automation
Set follow-up reminders
"Remind patient to retest HbA1c in 8 weeks." Doctor sets it once → patient gets WhatsApp reminder 2 days before + 1 day before.
Lost-to-follow-up detection
If patient doesn't return within expected window → doctor dashboard shows "Ramesh Patel — 6 weeks overdue for follow-up." Optional auto-WhatsApp.
Clinic analytics
How many patients retained, avg follow-up rate, most common conditions, alert frequency. For clinic billing plan.

SwasthAI Engine
SwasthAI Intelligence Layer
The AI brain that makes both portals valuable. Every service runs asynchronously. Patient uploads a document → engine processes in background → results appear within 30 seconds.
1. Document Extraction Service
Claude vision API (images)
JPG/PNG photos of lab reports, handwritten prescriptions. Claude reads the image directly — no OCR service needed.
Claude text API (PDF/WhatsApp)
PDF text extraction → Claude structured output. WhatsApp forwards → channel normalization → Claude extraction.
Output schema
labValues[], medications[], diagnoses[], followUpInstructions[], documentType, language, confidence. Stored in extractedData JSON column.
Multilingual
Claude handles Hindi, Gujarati, Tamil, Bengali, Marathi natively. No separate NLP layer. No pre-processing required.
Correction loop
Patient can correct any value → correction stored with original extraction → doctor brief shows corrected value + "patient corrected" flag.
lib/document-ingestion.tsworkers/extraction-worker.ts
2. Clinical Intelligence Pipeline
1
Lab normalization (ICMR/WHO)
Age + sex adjusted ranges. Child/adult/elderly tiers. 50+ tests. Output: NORMAL / LOW / HIGH / CRITICAL per value. File: lib/clinical-normalization.ts (549 lines — done)
2
Risk scoring
Diabetes, CKD, CVD, liver, thyroid risk. 0–100 score with contributing factor explanations. File: lib/risk-scoring.ts (done)
3
Trend detection
Single-lab: IMPROVING/STABLE/WORSENING with acceleration detection. Multi-lab: metabolic syndrome patterns, early CKD patterns. File: lib/trend-detection.ts (done)
4
Alert generation
CRITICAL_LAB_VALUE, ABNORMAL_TREND, MEDICATION_CONTRAINDICATION. 3-tier severity. Deduplication. WhatsApp trigger. File: lib/risk-store.ts (done)
5
Duplicate test detection
Same test within 45 days → patient warning + recommendation. File: lib/duplicate-test-store.ts (done)
6
Doctor brief generation
Claude API with specialty-aware prompt. 1-page narrative. Urgent flags, trends, medication review, suggested discussion points. Cached 1 hour. File: lib/product-engine.ts → upgrade to AI
3. Symptom Checker (Patient-facing AI)
Natural language input
Patient types/speaks in any language: "Teen din se chakkar aa raha hai" (Hindi: "dizzy for 3 days"). Claude responds in same language.
Context-aware
Claude receives patient's conditions, current medications, recent lab values before answering. Response is personalized, not generic.
Urgency triage
Output: urgency level (see doctor now / within a week / monitor at home) + reasons + relevant warning symptoms to watch.
Safety constraint: SwasthAI never diagnoses. System prompt always appended: "You are a health information assistant. Never provide a diagnosis. Always end with: consult your doctor if symptoms persist or worsen."

4. Agentic Pipeline (Multi-step)
Document uploaded → queue job
→ Step 1: extractDocumentContent() — Claude API
→ Step 2: deduplicateTests() — check against history
→ Step 3: normalizeExtractedLabs() — ICMR ranges
→ Step 4: computeRiskScores() — all conditions
→ Step 5: detectTrends() — delta from previous

Part 2 — Private Healthcare Payments (Umbra Protocol)
Problem: Paying an HIV clinic, a therapist, or an addiction specialist via UPI creates a visible transaction trail. In India, this can affect employment, insurance, and family relationships. Umbra Protocol solves this.
How Umbra works
1
Provider registers their stealth key pair
Provider generates spending + viewing key pair once. Registers on SwasthAI. Their real wallet is never public.
2
Patient selects provider + amount
SwasthAI generates a one-time stealth address for this payment. Uses ephemeral keypair + provider's public key → ECDH → stealth address.
3
Patient sends crypto to stealth address
SOL or USDC sent to stealth address on Solana. On-chain record: "Wallet A → Address B: 0.014 SOL." No names. No category. Nothing else.
4
Provider scans and claims
Provider uses their viewing key + SwasthAI scanning service to find payments addressed to them. Claims to their actual wallet. Patient's address never linked to provider's address.
5
Patient stores viewing key
Patient receives a viewing key (base64 string). This proves: amount + "Healthcare Service" (generic category). Used for insurance reimbursement.
6
Insurance proof on demand
Patient provides viewing key + payment ID → SwasthAI generates redacted proof PDF: amount, date, "Healthcare Service." Provider name never revealed.
